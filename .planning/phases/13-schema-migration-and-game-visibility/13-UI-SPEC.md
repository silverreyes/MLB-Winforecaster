# Phase 13 -- UI Design Contract

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding-block |
| sm | 8px | Compact element spacing, badge padding-inline |
| md | 16px | Default element spacing, card section padding |
| lg | 24px | Section padding, card header padding |
| xl | 32px | Grid gap between cards |
| 2xl | 48px | Grid top margin, major section breaks |
| 3xl | 64px | Page-level spacing (unused in this phase) |

Exceptions: none -- all Phase 13 elements use the existing scale from `index.css :root`.

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body | 16px | 400 | 1.5 | DM Sans (`--font-ui`) |
| Label / Badge | 12px | 600 | 1.3 | DM Sans (`--font-ui`) |
| Heading | 20px | 600 | 1.2 | DM Sans (`--font-ui`) |
| Hero Number | 32px | 400 | 1.0 | DM Mono (`--font-data`) |

Phase 13 new elements use:
- **StatusBadge text**: 12px weight 600 line-height 1.3 (`--font-ui`) -- matches existing SpBadge and EdgeBadge pattern.
- **Stub card matchup**: 16px weight 400 line-height 1.5 (`--font-ui`) -- same as existing `.matchup` class.
- **Stub card game time**: inherits existing `.gameTime` styles (defined in `GameCard.module.css`); no new size token introduced.

No new typography values introduced.

### New color tokens for Phase 13

No new CSS custom properties are added. StatusBadge colors use existing tokens:

| Badge State | Text Color | Background | Source |
|-------------|-----------|------------|--------|
| PRE-GAME | `--color-text-secondary` (`#8A8A9A`) | `rgba(138, 138, 154, 0.12)` | Neutral -- game not started; same muted pattern as model labels |
| LIVE | `--color-edge-green` (`#22C55E`) | `rgba(34, 197, 94, 0.12)` | Matches existing BUY YES badge -- green signals active/positive state |
| FINAL | `--color-text-secondary` (`#8A8A9A`) | `rgba(138, 138, 154, 0.12)` | Same as PRE-GAME -- game is resolved, no attention needed |
| POSTPONED | `--color-accent-muted` (`#D97706`) | `rgba(217, 119, 6, 0.15)` | Matches existing SP TBD badge -- amber signals caution/anomaly |

Accent reserved for: ensemble hero probability number, clock time display, "Last updated" timestamp. Never used for status badges.

### Modified Components

#### GameCard

| Change | Detail |
|--------|--------|
| Props | Change from `{ game: GameGroup; isStale: boolean }` to `{ game: GameResponse; isStale: boolean }` |
| StatusBadge placement | Rendered inside `.headerRow`, immediately after the matchup line (before `.spRow`). Sits between matchup text and SP info. |
| Stub card behavior | When `game.prediction` is `null`: render header (matchup + time + badge + SP row showing TBD for both) and omit the `.predictionBody` and `KalshiSection` entirely. The card shell (`.card` class) is the same. No graying, no dimming, no placeholder text in the probability area -- simply absent. |
| game_time source | Use `game.game_time` (from `GameResponse`) instead of looking inside prediction objects |

#### GameCardGrid

| Change | Detail |
|--------|--------|
| Props | Change from `{ games: GameGroup[]; isStale: boolean }` to `{ games: GameResponse[]; isStale: boolean }` |
| React key | Change from `${game.away_team}-${game.home_team}` to `game.game_id` (number, always unique, handles doubleheaders) |

## Interaction States

### Stub Card (game with no prediction)

| State | Visual |
|-------|--------|
| Default | Card shell with matchup, game time, StatusBadge, SP row (both TBD). No prediction body. No Kalshi section. Card height is naturally shorter. |
| Stale overlay | Same 0.45 opacity reduction applied to entire grid (existing `.stale` class). No per-card stale treatment. |

### Prediction Card (game with prediction)

| State | Visual |
|-------|--------|
| Default | Existing card layout with StatusBadge added after matchup line. All prediction columns and Kalshi section render as before. |
| Warning strip | Existing amber SP-change warning strip renders above header row, same as current behavior. |

| PRE_GAME, no prediction | PRE-GAME (gray) | Stub card (header only) |
| LIVE, has prediction | LIVE (green) | Full prediction card |
| LIVE, no prediction | LIVE (green) | Stub card (header only) |
| FINAL, has prediction | FINAL (gray) | Full prediction card |
| FINAL, no prediction | FINAL (gray) | Stub card (header only) |
| POSTPONED, has prediction | POSTPONED (amber) | Full prediction card |

## Layout Contract

Primary visual anchor: ensemble hero probability number (Hero Number, 32px DM Mono) on prediction cards; matchup heading (16px) on stub cards.

### Card placement of StatusBadge

```
+---------------------------------------+
| [Warning strip if SP changed]         |  <- existing, optional
+---------------------------------------+
| Matchup: Away @ Home                  |  <- .matchup (16px)
| 7:05 PM ET                            |  <- .gameTime styles
| [PRE-GAME]                            |  <- NEW: StatusBadge
| AwayPitcher | HomePitcher              |  <- .spRow (12px)
+---------------------------------------+
| Prediction body (if prediction)       |  <- .predictionBody
| ... or ABSENT (if stub card)          |
+---------------------------------------+
| Kalshi section (if prediction)        |  <- KalshiSection
| ... or ABSENT (if stub card)          |
+---------------------------------------+
```

The StatusBadge sits on its own line between the game time and SP row. It uses `margin: var(--space-xs) 0 0` (4px top margin, matching `.gameTime` and `.spRow` spacing pattern within the header).
