<objective>
Extend pitcher game log extraction to include K/BB/HR/numberOfPitches (SP-07, SP-08) and add computation functions for 30-day rolling FIP, pitch count from last start, and days rest.

Purpose: The v1 game log cache stores only 3 columns (date, innings_pitched, earned_runs). SP-07 requires strikeouts, base_on_balls, and home_runs for FIP computation. SP-08 requires numberOfPitches for pitch count feature and game dates for days-rest calculation. The MLB Stats API already returns all these fields -- the v1 code simply discarded them. This plan extends the parser and adds versioned caching (v2) so existing cached data is not silently reused.

Output: Extended `sp_recent_form.py` with v2 game log extraction, `compute_rolling_fip_bulk()`, `compute_pitch_count_and_rest_bulk()`, and comprehensive test suite.
</objective>

    **2. Add `_fetch_pitcher_game_log_v2()` to `src/features/sp_recent_form.py`:**

    Add a new function (do NOT modify the existing `_fetch_pitcher_game_log` -- preserve it for v1 compatibility):

    ```python
    V2_GAME_LOG_COLS = [
        "date", "innings_pitched", "earned_runs",
        "strikeouts", "base_on_balls", "home_runs",
        "number_of_pitches", "games_started",
    ]

    LEAGUE_AVG_PITCH_COUNT = 93  # League average pitches per start (imputation)
    MAX_DAYS_REST = 7            # Cap days rest at 7 (anything longer is treated as 7)

    def _fetch_pitcher_game_log_v2(player_id: int, season: int) -> pd.DataFrame:
        """Return per-game pitching stats with extended fields for one pitcher in one season.

        Cached as pitcher_game_log_v2_{season}_{player_id} (distinct from v1 cache).
        Columns: date, innings_pitched, earned_runs, strikeouts, base_on_balls,
                 home_runs, number_of_pitches, games_started.
        """
        key = f"pitcher_game_log_v2_{season}_{player_id}"
        parquet_path = f"mlb_api/pitcher_game_log_v2_{season}_{player_id}.parquet"

        if is_cached(key):
            df = read_cached(key)
            df["date"] = pd.to_datetime(df["date"])
            return df

        try:
            result = statsapi.get(
                "person",
                {
                    "personId": player_id,
                    "hydrate": f"stats(type=gameLog,group=pitching,season={season})",
                },
            )
            people = result.get("people", [])
            if not people:
                return pd.DataFrame(columns=V2_GAME_LOG_COLS)

            stats_list = people[0].get("stats", [])
            if not stats_list:
                return pd.DataFrame(columns=V2_GAME_LOG_COLS)

            splits = stats_list[0].get("splits", [])
            rows = []
            for s in splits:
                stat = s.get("stat", {})
                rows.append({
                    "date": pd.to_datetime(s.get("date")),
                    "innings_pitched": _parse_ip(stat.get("inningsPitched", "0.0")),
                    "earned_runs": int(stat.get("earnedRuns", 0)),
                    "strikeouts": int(stat.get("strikeOuts", 0)),
                    "base_on_balls": int(stat.get("baseOnBalls", 0)),
                    "home_runs": int(stat.get("homeRuns", 0)),
                    "number_of_pitches": int(stat.get("numberOfPitches", 0)),
                    "games_started": int(stat.get("gamesStarted", 0)),
                })
            df = pd.DataFrame(rows) if rows else pd.DataFrame(columns=V2_GAME_LOG_COLS)
            save_to_cache(df, key, parquet_path, season)
            return df

        except Exception as e:
            logger.debug("No v2 game log for player %d season %d: %s", player_id, season, e)
            return pd.DataFrame(columns=V2_GAME_LOG_COLS)
    ```

    def compute_rolling_fip_bulk(
        game_dates: list[str],
        season: int,
        sp_names: set[str] | None = None,
    ) -> dict[str, pd.DataFrame]:
        """Compute 30-day rolling raw FIP for each pitcher on each game date.

        Raw FIP = ((13*HR) + (3*BB) - (2*K)) / IP
        No cFIP constant needed -- cancels in home-away differentials.

        Returns: dict mapping date_str -> DataFrame with columns [Name, FIP].
        """
    ```

    Implementation follows the same pattern as `fetch_sp_recent_form_bulk()`:
    - Build pitcher name -> ID map via `_get_pitcher_id_map(season)`
    - Filter to `sp_names` if provided
    - Fetch game logs via `_fetch_pitcher_game_log_v2()` (not v1)
    - For each date, slice each pitcher's log to 30-day window `[date - 31 days, date)`
    - Aggregate: `sum_k`, `sum_bb`, `sum_hr`, `sum_ip` from the window
    - If `sum_ip > 0`: `raw_fip = ((13 * sum_hr) + (3 * sum_bb) - (2 * sum_k)) / sum_ip`
    - Return `{"Name": name, "FIP": raw_fip}` rows

    def compute_pitch_count_and_rest_bulk(
        game_dates: list[str],
        season: int,
        sp_names: set[str] | None = None,
    ) -> dict[str, pd.DataFrame]:
        """Compute pitch count from last start and days rest for each pitcher.

        Returns: dict mapping date_str -> DataFrame with columns
                 [Name, pitch_count_last, days_rest].
        pitch_count_last: numberOfPitches from most recent start before this date.
            Imputed to 93 (LEAGUE_AVG_PITCH_COUNT) if no prior start.
        days_rest: integer days since last start, capped at MAX_DAYS_REST (7).
            Set to MAX_DAYS_REST if no prior start found.
        """
    ```

    Implementation:
    - Same setup pattern (pitcher name -> ID, fetch v2 game logs)
    - For each date and pitcher, find the most recent start BEFORE this date:
      `prior_starts = log[(log["date"] < date_dt) & (log["games_started"] >= 1)]`
    - If `prior_starts` non-empty:
      - `last_start = prior_starts.iloc[-1]` (log is sorted by date)
      - `pitch_count_last = last_start["number_of_pitches"]`
      - `days_rest = min((date_dt - last_start["date"]).days, MAX_DAYS_REST)`
    - If empty (cold start):
      - `pitch_count_last = LEAGUE_AVG_PITCH_COUNT` (93)
      - `days_rest = MAX_DAYS_REST` (7)
