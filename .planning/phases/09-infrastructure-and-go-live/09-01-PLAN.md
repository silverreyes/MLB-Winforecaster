**Dockerfile** -- Multi-stage build with 3 stages:

Stage 1 (frontend-build):
- `FROM node:20-slim AS frontend-build`
- `WORKDIR /app`
- Copy `frontend/package.json` and `frontend/package-lock.json` first (layer cache)
- `RUN npm ci`
- Copy `frontend/` directory
- `RUN npm run build`
- Produces `/app/dist/` containing the built SPA

Stage 2 (python-deps):
- `FROM python:3.11-slim-bookworm AS python-deps`
- Install build dependencies: `apt-get update && apt-get install -y --no-install-recommends gcc libpq-dev && rm -rf /var/lib/apt/lists/*`
- Copy `requirements.txt`
- `RUN pip install --no-cache-dir -r requirements.txt gunicorn`

Stage 3 (runtime):
- `FROM python:3.11-slim-bookworm`
- Install runtime dependencies: `apt-get update && apt-get install -y --no-install-recommends libpq5 tzdata && rm -rf /var/lib/apt/lists/*`
- `WORKDIR /app`
- Copy Python packages from python-deps: `COPY --from=python-deps /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages`
- Copy Python binaries from python-deps: `COPY --from=python-deps /usr/local/bin /usr/local/bin`
- Copy built frontend: `COPY --from=frontend-build /app/dist ./frontend/dist`
- Copy application source: `COPY src/ ./src/`, `COPY api/ ./api/`, `COPY scripts/ ./scripts/`
- Copy pyproject.toml for editable install metadata: `COPY pyproject.toml .`
- Install the project package in non-editable mode: `RUN pip install --no-cache-dir --no-deps .`
- Default CMD: `CMD ["gunicorn", "api.main:app", "-k", "uvicorn.workers.UvicornWorker", "--bind", "0.0.0.0:8000", "--workers", "2", "--timeout", "120"]`

```yaml
    volumes:
      - ./models/artifacts:/app/models/artifacts:ro

  worker:
    build: .
    command: ["python", "scripts/run_pipeline.py"]
    mem_limit: 1024m
    environment:
      DATABASE_URL: postgresql://mlb:${POSTGRES_PASSWORD}@db:5432/mlb_forecaster
      KALSHI_API_KEY: ${KALSHI_API_KEY:-}
    volumes:
      - ./models/artifacts:/app/models/artifacts:ro
    depends_on:
      db:
        condition: service_healthy
    restart: unless-stopped

  mlb_pgdata:
```

CRITICAL: Use `mem_limit` (NOT `deploy.resources.limits.memory`) because `deploy` is ignored by `docker compose up` in non-swarm mode.

CRITICAL: `DATABASE_URL` uses `db` as host (Docker service name), NOT `localhost`. The existing db.py defaults to localhost, which is overridden by this environment variable inside the container.

CRITICAL: `models/artifacts` is bind-mounted read-only (`:ro`). Model artifacts are in .gitignore and not copied into the image -- they must exist on the VPS filesystem.

**scripts/memory_audit.sh** -- Pre-deploy memory gate (INFRA-01 hard gate). This script runs ON THE VPS before deploying the MLB stack:

```bash
#!/bin/bash
# Pre-deploy memory audit for MLB Win Forecaster
# Run on VPS BEFORE deploying: bash scripts/memory_audit.sh
# This is the INFRA-01 hard gate -- deploy is blocked if headroom < 1GB

set -e

echo "=== VPS Memory Audit ==="
echo ""

echo "System RAM:"
free -h | head -2
echo ""

echo "Current Docker container memory usage:"
docker stats --no-stream --format "table {{.Name}}\t{{.MemUsage}}\t{{.MemPerc}}" 2>/dev/null || echo "(no running containers)"
echo ""

echo "Projected MLB stack additions:"
echo "  api:    512MB"
echo "  worker: 1024MB"
echo "  db:     512MB"
echo "  Total:  2048MB (2GB)"
echo ""

USED_MB=$(free -m | awk '/Mem:/ {print $3}')
TOTAL_MB=$(free -m | awk '/Mem:/ {print $2}')
AVAILABLE_MB=$((TOTAL_MB - USED_MB))
NEEDED_MB=2048
REMAINING_MB=$((AVAILABLE_MB - NEEDED_MB))

echo "Currently used:    ${USED_MB}MB"
echo "Total RAM:         ${TOTAL_MB}MB"
echo "Available:         ${AVAILABLE_MB}MB"
echo "MLB stack needs:   ${NEEDED_MB}MB"
echo "Remaining after:   ${REMAINING_MB}MB"
echo ""

if [ "$REMAINING_MB" -lt 1000 ]; then
    echo "DEPLOY GATE: FAIL -- Less than 1GB headroom remaining after deploy."
    echo "Action required: Free memory or reduce container limits before deploying."
    exit 1
else
    echo "DEPLOY GATE: PASS -- Sufficient memory headroom (${REMAINING_MB}MB remaining)."
    exit 0
fi
```

**scripts/backup_postgres.sh** -- Daily pg_dump backup with 7-day retention (INFRA-04). Called by cron on VPS:

```bash
#!/bin/bash
# Daily Postgres backup for MLB Win Forecaster
# Cron entry: 0 3 * * * /opt/mlb-winforecaster/scripts/backup_postgres.sh >> /var/log/mlb_backup.log 2>&1

set -e

BACKUP_DIR="/opt/backups/mlb"
CONTAINER_NAME="mlb-winforecaster-db-1"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/mlb_forecaster_${TIMESTAMP}.sql.gz"

mkdir -p "$BACKUP_DIR"

echo "$(date): Starting backup..."

# pg_dump from the running Postgres container, pipe through gzip
docker exec "$CONTAINER_NAME" pg_dump -U mlb mlb_forecaster | gzip > "$BACKUP_FILE"

# Verify backup is non-empty
if [ $? -eq 0 ] && [ -s "$BACKUP_FILE" ]; then
    SIZE=$(du -h "$BACKUP_FILE" | cut -f1)
    echo "$(date): Backup successful: $BACKUP_FILE ($SIZE)"
else
    echo "$(date): ERROR: Backup failed or produced empty file"
    rm -f "$BACKUP_FILE"
    exit 1
fi

# Remove backups older than 7 days
DELETED=$(find "$BACKUP_DIR" -name "mlb_forecaster_*.sql.gz" -mtime +7 -delete -print | wc -l)
echo "$(date): Retention cleanup complete ($DELETED old backups removed)"
```

Both scripts must have executable permissions. Run `chmod +x scripts/memory_audit.sh scripts/backup_postgres.sh` after creating them.

IMPORTANT: The container name `mlb-winforecaster-db-1` is derived from the project directory name + service name + instance number. This is the Docker Compose default naming convention. If the project is cloned to a different directory on the VPS, the container name will differ. The backup script uses this name in the `docker exec` command.
