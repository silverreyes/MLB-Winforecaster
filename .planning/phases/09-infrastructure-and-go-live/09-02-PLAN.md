Create directory `nginx/` and file `nginx/mlbforecaster.conf`.

This is a REFERENCE TEMPLATE that will be copied to `/etc/nginx/sites-available/` (or `/etc/nginx/conf.d/`) on the VPS. The file includes comments explaining the deployment steps.

Content:

```nginx
# MLB Win Forecaster -- Nginx reverse proxy
# Deployment steps:
#   1. Copy to /etc/nginx/sites-available/mlbforecaster.silverreyes.net
#   2. ln -s /etc/nginx/sites-available/mlbforecaster.silverreyes.net /etc/nginx/sites-enabled/
#   3. nginx -t  (validate config)
#   4. systemctl reload nginx
#   5. certbot --nginx -d mlbforecaster.silverreyes.net  (issue SSL cert)
#   6. certbot renew --dry-run  (verify renewal works)

server {
    listen 80;
    server_name mlbforecaster.silverreyes.net;

    location / {
        proxy_pass http://127.0.0.1:8082;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
    }
}
# After running certbot --nginx, SSL directives will be auto-added above.
```

This listens on port 80 for the subdomain and proxies to the Docker Compose api service on port 8082. Certbot's `--nginx` plugin will automatically modify this file to add the SSL `listen 443 ssl` block, redirect HTTP to HTTPS, and add certificate paths.

The `proxy_http_version 1.1` and empty `Connection` header enable HTTP keep-alive between Nginx and the upstream, improving performance for repeated API polling requests.
