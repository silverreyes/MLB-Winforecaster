<objective>
Fix RETRY: APScheduler pipeline jobs fail immediately on a 503 or connection timeout from the MLB Stats API with no retry.

Root cause: create_scheduler() adds run_pipeline directly as the job function. Any exception (including transient 503s or timeouts from the statsapi/urllib layer) causes the job to fail with no retry.

Fix: add run_pipeline_with_retry() to scheduler.py. It calls run_pipeline; if a urllib.error.HTTPError with code 503 or a urllib.error.URLError/socket.timeout (connection timeout) is raised, it logs a warning, sleeps 15 minutes, and calls run_pipeline once more. Any second failure or any other exception type propagates normally. Replace run_pipeline with run_pipeline_with_retry in all three scheduler jobs.

Scope: do NOT modify runner.py. Retry logic lives entirely in scheduler.py.

The statsapi library uses urllib under the hood, so the exception types to catch are:
- urllib.error.HTTPError (check .code == 503)
- urllib.error.URLError (connection refused, DNS failure, timeout wrapped by urllib)
- socket.timeout (raw socket timeout that may escape urllib wrapping)
</objective>

    ```python
    RETRY_SLEEP_SECONDS = 900  # 15 minutes


    def run_pipeline_with_retry(version: str, artifacts: dict, pool) -> None:
        """Run a pipeline job with one retry on transient MLB Stats API errors.

        Retries once (after RETRY_SLEEP_SECONDS) on:
        - urllib.error.HTTPError with code 503 (service unavailable)
        - urllib.error.URLError (connection timeout, DNS failure, etc.)
        - socket.timeout (raw socket timeout)

        All other exception types propagate immediately without retry.
        If the retry also fails, its exception propagates normally.

        Args:
            version: 'pre_lineup', 'post_lineup', or 'confirmation'.
            artifacts: All 6 loaded model artifacts.
            pool: psycopg ConnectionPool.
        """
        try:
            run_pipeline(version, artifacts, pool)
        except urllib.error.HTTPError as exc:
            if exc.code != 503:
                raise
            logger.warning(
                "MLB Stats API returned 503 for %s pipeline — retrying in %d minutes",
                version,
                RETRY_SLEEP_SECONDS // 60,
            )
            time.sleep(RETRY_SLEEP_SECONDS)
            run_pipeline(version, artifacts, pool)
        except (urllib.error.URLError, socket.timeout) as exc:
            logger.warning(
                "MLB Stats API connection error (%s) for %s pipeline — retrying in %d minutes",
                type(exc).__name__,
                version,
                RETRY_SLEEP_SECONDS // 60,
            )
            time.sleep(RETRY_SLEEP_SECONDS)
            run_pipeline(version, artifacts, pool)
    ```
