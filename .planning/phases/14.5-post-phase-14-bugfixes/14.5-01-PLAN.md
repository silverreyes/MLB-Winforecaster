<objective>
Fix BUG-A: "Last updated" in the header shows the page-load time (games-response generated_at = datetime.now() at request time) instead of the pipeline run time.

Root cause: App.tsx passes displayedTimestamp = data?.generated_at (set to datetime.now() server-side) to Header as lastUpdated. The /predictions/latest-timestamp endpoint already exists and is already polled by useLatestTimestamp, but its value is only used for the hasNewPredictions comparison, not for the header display.

Fix: wire timestampData?.timestamp from useLatestTimestamp to Header's lastUpdated and the isStale check. Fix hasNewPredictions to use a ref tracking the initial pipeline timestamp so the banner fires correctly when a new run arrives after page load.

Output: Header shows real pipeline run time; isStale and hasNewPredictions behave correctly.
</objective>
