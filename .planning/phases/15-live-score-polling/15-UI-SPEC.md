| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Body / matchup | 16px | 400 | 1.5 | `--font-ui` (DM Sans) |
| Score row | 16px | 600 | 1.5 | `--font-data` (DM Mono) |

Phase 15-specific notes:
- Score row uses `--font-data` (DM Mono) at 16px/600 for numerical score alignment and readability.
- Inning + outs label within the score row uses `--font-ui` at 13px/400 -- the same meta style as game time.
- Expanded section batter names use 13px/400 `--font-ui`. Batter stats (AVG/OPS) use 13px/400 `--font-data`.

Source: Established sizes in existing `GameCard.module.css` and `EmptyState.module.css`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0A0A0F` (`--color-bg`) | Page background |
| Secondary (30%) | `#12121A` (`--color-surface`) | Card background, expanded section background |
| Accent (10%) | `#F59E0B` (`--color-accent`) | Score row text, occupied base fill, expand chevron |
| Border | `#1E1E2A` (`--color-border`) | Card border, empty base outline, score row top/bottom border |
| Text primary | `#E8E8ED` (`--color-text-primary`) | Team names in score row |
| Text secondary | `#8A8A9A` (`--color-text-secondary`) | Inning text, batter stats labels, on-deck batter name |
| Live green | `#22C55E` (`--color-edge-green`) | LIVE status badge (existing, no change) |

Accent reserved for:
1. Score row score digits (amber monospace numerals)
2. Occupied base diamond fill (solid amber)
3. Expand/collapse chevron indicator on score row

Score row background: `rgba(245, 158, 11, 0.08)` -- a very subtle amber tint over card surface to visually distinguish the live score strip from the rest of the card. This follows the existing pattern of `rgba(217, 119, 6, 0.15)` used by the warning strip, but at lower opacity to avoid competing with it.

Empty base diamond: stroke `--color-border` (`#1E1E2A`), no fill (transparent).
Occupied base diamond: fill `--color-accent` (`#F59E0B`), no stroke.

Source: CONTEXT.md locked decisions and existing `StatusBadge.module.css` color patterns.

### Score Row Text Format

Pattern: `{AWAY} {away_score} - {HOME} {home_score}  *  {Top|Bot} {N}{suffix}  *  {outs} out`

Examples:
- `NYY 3 - BOS 1  *  Top 7th  *  2 out`
- `LAD 0 - SF 0  *  Bot 1st  *  0 out`
- `HOU 5 - TEX 4  *  Top 9th  *  1 out`

The bullet separator (`*`) is rendered as a centered dot character (U+00B7 `·`) in `--color-text-secondary`.

Inning suffix rules: 1st, 2nd, 3rd, 4th-9th+th. "Top" / "Bot" for inning half.
Outs: always singular "out" (not "outs") -- e.g., "0 out", "1 out", "2 out".

### Expanded Detail Content Format

Count row: `B: {balls}  S: {strikes}  O: {outs}` -- monospace, all single digits.

Batter row:
- Line 1 (batterName): `{First initial}. {Last name}` -- e.g., `J. Turner`
- Line 2 (batterStats): `.{AVG} / .{OPS}` -- e.g., `.287 / .821`

On-deck row:
- Line 1 (onDeckLabel): `On deck:`
- Line 2 (onDeckName): `{First initial}. {Last name}` -- e.g., `R. Devers`

If `current_batter` is null (between-innings or data unavailable): show `--` in place of batter name, omit stats line.
If `on_deck_batter` is null: omit the entire on-deck section.
If batter AVG/OPS is null: show name only, omit stats line.

## Bases Diamond SVG Specification

Dimensions: 80px x 80px viewBox (`viewBox="0 0 80 80"`).

Diamond orientation: Home plate at bottom center, 1B at right, 2B at top, 3B at left.

Base coordinates (center points of each diamond shape):
- Home plate: (40, 72) -- decorative only, always rendered as outline
- 1st base: (64, 40)
- 2nd base: (40, 8)
- 3rd base: (16, 40)

Each base is a small rotated square (diamond) with a side length of ~12px:

```svg
<svg viewBox="0 0 80 80" width="80" height="80" xmlns="http://www.w3.org/2000/svg">
  <!-- Basepaths (decorative lines connecting bases) -->
  <line x1="40" y1="72" x2="64" y2="40" stroke="#1E1E2A" stroke-width="1" />
  <line x1="64" y1="40" x2="40" y2="8" stroke="#1E1E2A" stroke-width="1" />
  <line x1="40" y1="8" x2="16" y2="40" stroke="#1E1E2A" stroke-width="1" />
  <line x1="16" y1="40" x2="40" y2="72" stroke="#1E1E2A" stroke-width="1" />

  <!-- Home plate (always outline) -->
  <polygon points="40,78 34,72 34,68 46,68 46,72"
           fill="none" stroke="#1E1E2A" stroke-width="1.5" />

  <!-- 1st base -->
  <rect x="58" y="34" width="12" height="12" rx="1"
        transform="rotate(45, 64, 40)"
        fill="{#F59E0B if runner_on_1b else 'none'}"
        stroke="#1E1E2A" stroke-width="1.5" />

  <!-- 2nd base -->
  <rect x="34" y="2" width="12" height="12" rx="1"
        transform="rotate(45, 40, 8)"
        fill="{#F59E0B if runner_on_2b else 'none'}"
        stroke="#1E1E2A" stroke-width="1.5" />

  <!-- 3rd base -->
  <rect x="10" y="34" width="12" height="12" rx="1"
        transform="rotate(45, 16, 40)"
        fill="{#F59E0B if runner_on_3b else 'none'}"
        stroke="#1E1E2A" stroke-width="1.5" />
</svg>
```

Occupied base: `fill="#F59E0B"` (`--color-accent`), `stroke="#1E1E2A"`.
Empty base: `fill="none"` (transparent), `stroke="#1E1E2A"`.

The SVG is rendered inline in `BasesDiamond.tsx` as a React component accepting `runner_on_1b`, `runner_on_2b`, `runner_on_3b` boolean props.

## Interaction Contract

### Expand / Collapse

| Property | Value |
|----------|-------|
| Trigger | Click anywhere on `.scoreRow` (entire row is clickable) |
| Affordance | Chevron character (U+25BE `v` or SVG down-arrow) at right edge of score row |
| Open indicator | Chevron rotates 180 degrees (points up) |
| Animation | `transform 150ms ease` on chevron only; expanded section has no height animation (instant show/hide via conditional render) |
| Default state | Collapsed |
| Persistence | Per-card `useState<boolean>(false)` -- resets on page navigation or data refresh |
| Scope | Only available on `game_status === 'LIVE'` cards; score row is not rendered for any other status |

### Polling Behavior

| Property | Value |
|----------|-------|
| Interval | 90 seconds (90000ms) |
| Gate condition | `viewMode === 'live'` AND `games.some(g => g.game_status === 'LIVE')` |
| When gate is false | `refetchInterval` returns `false` (no polling) |
| staleTime | 55000ms (unchanged from current) |
| Data source | Existing `/games/{date}` endpoint with live score fields populated server-side |

### State Transitions

| From | To | UI Effect |
|------|----|-----------|
| PRE_GAME | LIVE | Score row appears between header and prediction body; status badge changes to green LIVE |
| LIVE | LIVE (data update) | Score row text updates in-place; if expanded, detail section updates in-place; expand state preserved |
| LIVE | FINAL | Score row disappears; expanded section disappears; status badge changes to gray FINAL; card returns to collapsed layout |
| LIVE (expanded) | FINAL | Same as above -- expanded section removed, card collapses |

When a game transitions from LIVE to FINAL mid-poll, the expand state is implicitly cleared because the score row (and its expand trigger) is conditionally rendered only for LIVE status.

## Accessibility Notes

- Score row has `role="button"` and `tabindex="0"` for keyboard access; responds to Enter and Space to toggle expand.
- Chevron has `aria-expanded="{true|false}"` reflecting the expand state.
- Expanded detail section has `aria-label="Live game details"`.
- Bases diamond SVG has `role="img"` and `aria-label` describing runner positions, e.g., `"Runners on first and third"` or `"Bases empty"`.
- Score values use `aria-live="polite"` so screen readers announce score updates without interrupting.
