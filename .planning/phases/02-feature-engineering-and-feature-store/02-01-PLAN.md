Create `src/features/game_logs.py` with two functions:

1. `fetch_team_game_log(season: int, team: str) -> pd.DataFrame`:
   - Cache key: `team_game_log_{season}_{team}` (e.g., `team_game_log_2023_NYY`)
   - Parquet path: `pybaseball/team_game_log_{season}_{team}.parquet`
   - If cached, return `read_cached(key)` immediately
   - Otherwise call `pybaseball.team_game_logs(season, team)` (import as `from pybaseball import team_game_logs as pybaseball_team_game_logs` for mock-friendly aliasing)
   - Add metadata columns: `team = team`, `season = season`, `is_shortened_season = (season == 2020)`
   - Call `save_to_cache(df, key, parquet_path, season)` and return
   - On exception, log error and re-raise (do NOT silently swallow)

NOTE: Callers always pass canonical team codes from ALL_TEAMS. pybaseball's team_game_logs() accepts these codes directly (verified in RESEARCH.md Pitfall 1). No normalize_team() call is needed here -- normalization is handled upstream by callers who source team codes from schedule data via normalize_team().

2. `fetch_all_team_game_logs(seasons: list[int], teams: list[str]) -> list[tuple[int, str]]`:
   - Returns a list of `(season, team)` tuples that failed; empty list on full success
   - Iterates all season x team combinations
   - For each: check `is_cached(f"team_game_log_{season}_{team}")` FIRST
   - If cached, skip (no delay needed -- instant return)
   - If not cached, call `fetch_team_game_log(season, team)`
   - After each UNCACHED fetch, sleep `time.sleep(2.0)` (RATE_LIMIT_DELAY constant at module level)
   - On exception: log error with `logger.error(f"Failed {team} {season}: {e}")`, append `(season, team)` to a `failures` list, sleep `RATE_LIMIT_DELAY * 2` (exponential backoff), continue to next (do NOT abort batch)
   - Return `failures` at end so callers can surface failures without log inspection
   - Log progress: `logger.info(f"Fetched {team} {season} ({fetched}/{total_uncached})")`
   - Use `logging.getLogger(__name__)` for logger
