# Phase 8 — UI Design Contract

> Visual and interaction contract for the MLB Win Forecaster dashboard at mlbforecaster.silverreyes.net. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-built components) |
| Icon library | none (use Unicode symbols: warning triangle, arrows, bullets only) |
| Data font | DM Mono (Google Fonts, 400 weight) |
| UI font | DM Sans (Google Fonts, 400 + 600 weights) |

**Font loading strategy:** Self-host via `@font-face` in `frontend/src/index.css`. Include `woff2` format only. Subset to Latin. Preload both regular weights in `index.html` via `<link rel="preload">`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Compact element spacing, model row gaps, accuracy strip vertical padding |
| md | 16px | Default element spacing, card padding, accuracy strip horizontal padding |
| lg | 24px | Card internal section breaks |
| xl | 32px | Grid gap between game cards |
| 2xl | 48px | Page header to content gap |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch target minimum is 44px height for the "New predictions available" refresh banner button (accessibility).

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Hero probability | DM Mono | 32px | 400 | 1.0 | Ensemble average percentage on game card |
| Heading | DM Sans | 20px | 600 | 1.2 | Page title ("Today's Predictions"), accuracy strip heading |
| Body | DM Sans | 16px | 400 | 1.5 | Page descriptions, banner messages, error state body copy |
| Data / Label | DM Mono / DM Sans | 12px | 400 / 600 | 1.3 | DM Mono 400: individual model probabilities (LR/RF/XGB), Kalshi price, edge magnitude, timestamps. DM Sans 600: section headings within cards ("MODELS", "KALSHI"), badge text, column headers. Font-family and weight differences maintain visual distinction within the 12px tier. |

**Weights used:** 400 (regular — all DM Mono data values/timestamps, DM Sans body) and 600 (bold — DM Sans headings and section labels only). Two weights total.

**Tabular figures:** DM Mono uses tabular figures by default. Apply `font-variant-numeric: tabular-nums` as a safety fallback on all containers rendering probability or price values so columns align vertically.

### State-Specific Color Rules

| State | Treatment |
|-------|-----------|
| Post-lineup card (primary) | Full opacity, amber hero number |
| Pre-lineup reference | Hero number at `opacity: 0.55`, label "PRE-LINEUP" in `#8A8A9A` |
| SP: TBD | Amber badge `#D97706` on `rgba(217, 119, 6, 0.15)` background; card NOT grayed out |
| SP may have changed | Amber warning strip across card top: `rgba(217, 119, 6, 0.15)` background with `#D97706` text |
| Stale (>3 hours) | Entire card gets `opacity: 0.45` overlay; staleness badge in `#6B7280` replaces normal timestamp |
| Offline/error | Card area shows error state; no card grid; error text in `#8A8A9A` on `#12121A` |
| BUY_YES edge | Badge: `#22C55E` text on `rgba(34, 197, 94, 0.12)` background |
| BUY_NO edge | Badge: `#EF4444` text on `rgba(239, 68, 68, 0.12)` background |
| NO_EDGE | No badge rendered; Kalshi price shown in `#8A8A9A` without edge indicator |

## Layout Contract

### Page Structure

```
+------------------------------------------------------------------+
|  HEADER: "MLB Win Forecaster" (heading)   "Last updated: ..." (ts)|
+------------------------------------------------------------------+
|  ACCURACY STRIP (full-width bar)                                  |
|  "XGB SP: 0.235 | RF SP: 0.234 | LR SP: 0.233 | Kalshi: 0.224" |
+------------------------------------------------------------------+
|  [NEW PREDICTIONS BANNER — conditional, amber background]         |
+------------------------------------------------------------------+
|  GAME CARD GRID                                                   |
|  +------------------+  +------------------+  +------------------+ |
|  | Game Card        |  | Game Card        |  | Game Card        | |
|  +------------------+  +------------------+  +------------------+ |
|  +------------------+  +------------------+  +------------------+ |
|  | Game Card        |  | Game Card        |  | Game Card        | |
|  +------------------+  +------------------+  +------------------+ |
+------------------------------------------------------------------+
```

**Grid:** CSS Grid, `grid-template-columns: repeat(auto-fill, minmax(340px, 1fr))`, gap `32px`. Max content width `1200px`, centered with `margin: 0 auto`. Page padding: `48px` top, `24px` sides.

**Responsive breakpoints:**
- >= 1200px: up to 3 columns
- 768px-1199px: 2 columns
- < 768px: 1 column, card padding reduces to `8px`, hero probability reduces to `28px`

### Game Card Anatomy

```
+------------------------------------------------------+
| [AMBER WARNING STRIP — only if sp_may_have_changed]  |
+------------------------------------------------------+
|  AWAY TEAM  @  HOME TEAM          [Game time / Final] |
|  Away SP Name     Home SP Name    [SP:TBD badge]     |
+------------------------------------------------------+
|  POST-LINEUP              |  PRE-LINEUP              |
|  Ensemble: 62.4%  (hero)  |  Ensemble: 58.1%  (muted)|
|                            |                          |
|  LR   61.2%               |  LR   57.8%              |
|  RF   63.1%               |  RF   58.0%              |
|  XGB  62.8%               |  XGB  58.6%              |
+------------------------------------------------------+
|  KALSHI                                               |
|  Price: 55c          [BUY YES +7.4pts]               |
+------------------------------------------------------+
```

**When only pre-lineup exists:** Pre-lineup occupies the full card width at full opacity (no split layout). Label reads "TEAM ONLY" instead of "PRE-LINEUP".

**When both versions exist:** Two-column split inside the card. Post-lineup takes left column at full opacity. Pre-lineup takes right column at reduced opacity (`0.55`).

### Accuracy Strip

Single horizontal bar below the header. Background `#12121A` with `1px` bottom border in `#1E1E2A`. Padding `8px 16px` (--space-sm vertical, --space-md horizontal). Content is a single line of DM Mono 12px values separated by `|` dividers in `#1E1E2A`. Label portion ("XGB SP:") in DM Sans 12px weight 600 `#8A8A9A`. Value portion (e.g., "0.235") in DM Mono 12px weight 400 `#F59E0B`.

Data source: pre-rendered from `models/artifacts/model_metadata.json` at build time or loaded statically. NOT a live API-04 call.

Brier scores to display (from model_metadata.json aggregate values):
- LR SP: 0.233
- RF SP: 0.234
- XGB SP: 0.235
- LR Team: 0.237
- RF Team: 0.238
- XGB Team: 0.240

Label each with its benchmark source: "2015-2024 backtest" for model scores. If a Kalshi 2025 comparison value is available, append it.

## Interaction Contract

### Polling (DASH-06)

| Behavior | Specification |
|----------|---------------|
| Poll endpoint | `GET /api/v1/predictions/latest-timestamp` |
| Poll interval | 60 seconds |
| Poll condition | `document.visibilityState === 'visible'` |
| Suspend on | `visibilitychange` to `'hidden'` |
| Resume on | `visibilitychange` to `'visible'` (immediate poll on resume) |
| New data detection | Polled timestamp > currently displayed timestamp |
| New data action | Show amber banner with "Load latest predictions" button |
| Banner click | Full data refetch from `GET /api/v1/predictions/today` (NOT a browser reload) |
| Implementation | React Query with `refetchInterval: 60000` and `refetchIntervalInBackground: false` on the timestamp query |

### Staleness (DASH-05)

| Behavior | Specification |
|----------|---------------|
| Staleness threshold | 3 hours since most recent prediction `created_at` |
| Visual treatment | All game cards receive `opacity: 0.45`; staleness text replaces normal "Last updated" |
| Staleness copy | "Data may be stale — last updated [timestamp]" in `#6B7280` |
| Check frequency | Evaluated on every render (compare `Date.now()` against displayed data timestamp) |

### Error State (DASH-07)

| Behavior | Specification |
|----------|---------------|
| Trigger | API fetch returns network error or non-2xx status |
| Visual treatment | Replace game card grid with centered error state block |
| Error block | `#12121A` background, `24px` padding, `1px` border `#1E1E2A`, max-width `480px` centered |
| Content | Heading "Dashboard offline" in DM Sans 20px 600 `#E8E8ED`, body text in DM Sans 16px 400 `#8A8A9A` |
| Last-known data | If cached data exists, show it with the staleness overlay + "Dashboard offline" badge in header area |
| No cached data | Show error block only; no game cards |
| Recovery | Continue polling; auto-recover when API responds; remove error state and show fresh data |

### Loading State

| Behavior | Specification |
|----------|---------------|
| Initial load | Skeleton cards: `#12121A` rectangles with subtle pulse animation (`opacity` oscillating `0.3` to `0.6` over `1.5s ease-in-out infinite`) |
| Skeleton count | 6 skeleton cards (typical MLB daily game count) |
| Skeleton shape | Match game card dimensions: `340px` min-width, `280px` height |
| Transition | Skeleton cards replaced by real cards on data arrival; no layout shift (same grid) |
