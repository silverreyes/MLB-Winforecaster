## Interaction Contracts

### DateNavigator Interactions

| Action | Result |
|--------|--------|
| Click left arrow | `selectedDate` decremented by 1 day. TanStack Query fetches `/games/{newDate}`. |
| Click right arrow | `selectedDate` incremented by 1 day. TanStack Query fetches `/games/{newDate}`. |
| Change date input value | `selectedDate` set to the input value. TanStack Query fetches `/games/{newDate}`. |
| Click "Today" button | `selectedDate` set to `todayDateStr()`. No-op if already on today. |
| Keyboard: left/right arrows while arrow button focused | Standard button activation (Enter/Space) -- no custom keyboard shortcuts on the container. |

### Polling Behavior by View Mode

| View Mode | Polling | Rationale |
|-----------|---------|-----------|
| `live` (today) | `refetchInterval: 60_000` (60s) | Today's predictions update as pipeline runs |
| `historical` (past) | `refetchInterval: false` | Past dates never change |
| `tomorrow` | `refetchInterval: false` | Tomorrow's schedule changes rarely; manual refresh sufficient |
| `future` | `refetchInterval: false` | Future schedule is static |

### Date Arithmetic Safety

All date arithmetic in the frontend MUST use noon-anchored Date construction to avoid timezone off-by-one:
```typescript
const d = new Date(selectedDate + 'T12:00:00');
d.setDate(d.getDate() + offset);
```
Never use `new Date('YYYY-MM-DD')` alone (parsed as UTC midnight, wrong day in US timezones).

### Stale "Today" After Midnight

When the server returns `view_mode: 'historical'` for a date the client believes is today, show an inline note in the DateNavigator: "A new day has started. Click Today to refresh." Rendered as a `<span>` with `font-size: 12px; color: var(--color-accent-muted);` next to the Today button.

Displayed as a badge on the card. No additional tooltip or explanation beyond the FutureDateBanner context.

### Error State Copy

No changes to ErrorState component for Phase 14. The existing "Dashboard offline" / "Unable to reach the predictions API" copy applies to all dates equally.

### Destructive Actions

None. Phase 14 contains no destructive actions (no deletes, no resets, no overwrites). No confirmation dialogs needed.

## States Matrix

| State | View Mode | What Renders |
|-------|-----------|-------------|
| Loading (first fetch) | any | SkeletonCard grid (6 cards) |
| Error (no cached data) | any | ErrorState ("Dashboard offline") |
| Error (has cached data) | any | Stale data with offline badge in Header |
| Empty (zero games) | live | EmptyState with "today" copy |
| Empty (zero games) | historical | EmptyState with past-date copy |
| Empty (zero games) | tomorrow | EmptyState with "tomorrow" copy |
| Empty (zero games) | future | EmptyState with future-date copy |
| Has games | live | GameCardGrid with full prediction data, polling active |
| Has games | historical | GameCardGrid with stored predictions, no polling |
| Has games, all SPs confirmed | tomorrow | FutureDateBanner + GameCardGrid with PRELIMINARY badges, SP names visible |
| Has games, some SPs TBD | tomorrow | FutureDateBanner + GameCardGrid with mix of PRELIMINARY and schedule-only cards |
| Has games | future | FutureDateBanner + GameCardGrid with schedule-only stub cards (header only, no prediction body) |
