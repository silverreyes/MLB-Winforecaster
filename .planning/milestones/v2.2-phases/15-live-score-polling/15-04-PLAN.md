  <what-built>
Complete live score polling system:
- Backend: LiveScoreData model, linescore cache (90s TTL), parse_linescore function, /games/{date} enrichment for LIVE games, live_poller_job (90s APScheduler, writes outcomes on Final), write_game_outcome targeting all prediction rows
- Frontend: LiveScoreData TypeScript interface, 90s dual-gate polling (today + live games exist), ScoreRow with expand/collapse (LIVE only), LiveDetail with bases diamond + pitch count + batter stats + on-deck, BasesDiamond inline SVG
  </what-built>
  <how-to-verify>
**Pre-deployment:**
1. Run `docker compose build && docker compose up -d` on VPS
2. Wait for containers to be healthy: `docker compose ps`

**Live Score Display (LIVE-01, LIVE-03-07):**
3. Navigate to the dashboard at the production URL
4. If games are currently in-progress (LIVE status):
   - Verify LIVE cards show a score row between the header and prediction body
   - Score row format: `{AWAY} {score} - {HOME} {score} * {Top|Bot} {N}th * {N} out`
   - Score row has amber-tinted background (subtle, not bright)
   - Click the score row to expand -- verify:
     a. Bases diamond SVG appears with runners highlighted in amber on occupied bases
     b. Pitch count row: `B: {N}  S: {N}  O: {N}`
     c. Current batter name with AVG/OPS stats
     d. On-deck batter name (or section hidden if null)
   - Click score row again to collapse
   - Verify PRE_GAME and FINAL cards do NOT have a score row or expand affordance

**Polling Behavior (LIVE-02):**
5. Open browser DevTools > Network tab
6. With today selected and LIVE games present: confirm `/api/games/` requests appear every ~90 seconds
7. Navigate to a past date: confirm polling stops (no more network requests)
8. Navigate back to today with no LIVE games (if all are PRE_GAME or FINAL): confirm no polling

**Outcome Write (LIVE-08):**
9. Check Docker logs for the worker container: `docker compose logs worker --tail 50`
10. Look for "Live score poller" log messages:
    - "no live or final games, skipping cycle" when no games are live
    - "wrote outcome for game {id}" if a game recently finished
11. If a game finished, verify in Postgres: `SELECT game_id, actual_winner, prediction_correct, reconciled_at FROM predictions WHERE reconciled_at IS NOT NULL LIMIT 5;`

**If no games are currently in-progress:**
- The LIVE score row will not appear (expected behavior)
- Verify the poller is running: check worker logs for "Live score poller" cycle messages
- Verify PRE_GAME cards have no score row or expand trigger
- Mark as approved with note "no LIVE games available for visual verification"
