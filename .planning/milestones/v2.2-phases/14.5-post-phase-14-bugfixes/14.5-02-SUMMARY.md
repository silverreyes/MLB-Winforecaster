**useEasternClock displays clock and next-update times in browser-local timezone via Intl.DateTimeFormat without timeZone property**

## Accomplishments
- useEasternClock.ts now uses localTimeFmt (no timeZone specified) for clock display, defaulting to the browser's timezone
- etRunHourToLocalDisplay function converts ET-scheduled pipeline run hours to the user's local timezone for next-update display
- Removed hardcoded ' ET' suffix from clock output and 'ET' labels from next-update strings
- dateFmt preserved in ET (America/New_York) for baseball-date context

## Decisions Made
- dateFmt keeps ET timezone for baseball-date context -- the header date reflects the "baseball day" which is always ET
- localTimeFmt omits the timeZone property so Intl.DateTimeFormat formats in whatever timezone the browser is set to
- etRunHourToLocalDisplay computes the offset between local and ET hours, then converts the ET run hour to local time for display
